# Automated Lead Management & Email Tracking System

Full-stack lead capture system with AI qualification, automated email delivery, open tracking, click tracking, and analytics.

## Architecture

```text
eval/
  backend/
    src/
      config/          MongoDB and environment configuration
      controllers/     MVC request handlers
      middleware/      validation and error middleware
      models/          Mongoose schemas
      routes/          Express route modules
      services/        Gemini and Nodemailer integrations
      utils/           async and tracking-token helpers
      app.js           Express app
      server.js        API bootstrap
    .env.example
    package.json
  frontend/
    src/
      api/             Axios client
      components/      Form, KPI cards, tables
      pages/           Dashboard
      App.jsx
      main.jsx
      styles.css
    .env.example
    package.json
```

## Database Schema

`Lead`

| Field | Type | Notes |
| --- | --- | --- |
| `fullName` | String | Required |
| `email` | String | Required, normalized |
| `phone` | String | Required |
| `company` | String | Optional |
| `requirement` | String | Required |
| `aiAnalysis.category` | String | Generated by Gemini |
| `aiAnalysis.priority` | `High`, `Medium`, `Low` | Generated by Gemini |
| `aiAnalysis.summary` | String | Generated by Gemini |
| `emailTracking.sent` | Boolean | Set after Nodemailer sends |
| `emailTracking.sentAt` | Date | Email send timestamp |
| `emailTracking.messageId` | String | SMTP provider message id |
| `emailTracking.open` | Object | Open timestamp, user agent, IP |
| `emailTracking.clicks[]` | Array | Destination, click timestamp, user agent, IP |

## API Design

Base URL: `http://localhost:5000/api`

| Method | Endpoint | Description |
| --- | --- | --- |
| `POST` | `/leads` | Capture lead, run Gemini analysis, send email |
| `GET` | `/leads` | List leads |
| `GET` | `/leads/:id` | Get one lead |
| `GET` | `/analytics` | Dashboard totals, chart data, recent leads |
| `GET` | `/tracking/open/:leadId.png?t=token` | Tracking pixel endpoint |
| `GET` | `/tracking/click/:leadId?t=token&url=...` | Track click and redirect |

## Services

Gemini receives the lead requirement and returns JSON:

```json
{
  "category": "CRM Automation",
  "priority": "High",
  "summary": "The lead needs a CRM workflow with fast follow-up automation."
}
```

Nodemailer sends a personalized HTML/text email including the user name, requirement, AI category, AI priority, AI summary, a signed trackable link, and a hidden tracking pixel.

## Setup

### Backend

```bash
cd backend
cp .env.example .env
npm install
npm run dev
```

Fill `backend/.env` with MongoDB Atlas, Gemini, and SMTP credentials.

For Gmail SMTP, use an app password instead of your normal Google password.

### Frontend

```bash
cd frontend
cp .env.example .env
npm install
npm run dev
```

Open `http://localhost:5173` for the public user form.

Client dashboard:

```text
http://localhost:5173/dashboard
```

## Environment Variables

Backend:

```env
PORT=5000
BASE_URL=http://localhost:5000
FRONTEND_URL=http://localhost:5173
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/lead-management
GEMINI_API_KEY=your_gemini_api_key
GEMINI_MODEL=gemini-1.5-flash
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_SECURE=false
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password
EMAIL_FROM_NAME=Lead Management System
EMAIL_FROM_ADDRESS=your_email@gmail.com
TRACKING_SECRET=replace-with-a-long-random-string
```

Frontend:

```env
VITE_API_BASE_URL=http://localhost:5000/api
```

## Notes

- If Gemini credentials are missing, the backend stores a safe fallback analysis so lead capture still works.
- If SMTP credentials are missing, the backend captures the lead but skips email sending.
- Tracking links are signed with `TRACKING_SECRET`.
- Email opens depend on the recipient email client loading remote images.
